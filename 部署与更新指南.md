## 适用范围

　　本文档用于 TechVenture（Node.js + Express + 静态页面）在比赛服务器上的日常更新流程：你在本地修复/改动后，如何安全、可重复地把新版上线到 `https://techventures1.online/`。

## 你当前的线上结构（已部署完成）

　　服务器端代码目录：`/srv/techventure`  
　　进程守护：`systemd` 服务 `techventure`  
　　反向代理：`nginx`（80 自动跳转 443；443 走证书）  
　　环境变量文件：`/etc/techventure/techventure.env`

## 常见故障与处理

### 1）`EADDRINUSE: 127.0.0.1:3780`（端口被占用）

　　现象：`sudo systemctl status techventure` 显示 failed，日志出现 `EADDRINUSE`。

　　处理思路：说明有另一份 Node 进程占用端口。按下面命令找到并结束占用进程，再重启服务：

```bash
ss -lntp | grep 3780 || true
sudo systemctl restart techventure
sudo systemctl status techventure --no-pager
```

### 2）改了代码但线上没变化

　　常见原因是服务器还没 `git pull`，或构建失败但你没注意到，或重启没成功。

　　按“服务器更新上线”的 1→3 步重新跑一遍，尤其确认：

```bash
cd /srv/techventure
git log -1 --oneline
sudo systemctl status techventure --no-pager
```

### 3）证书/HTTPS 问题

　　证书自动续期由 Certbot 定时任务处理。遇到异常先检查证书状态：

```bash
sudo certbot certificates
sudo certbot renew --dry-run
```

　　比赛过程中，所有队伍/轮次/提交都会写入：`/var/lib/techventure/techventure.json`。这是最重要的数据文件。
