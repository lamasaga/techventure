TechVenture 现场服务（局域网）

1. 安装依赖（纯 JavaScript 持久化，无需编译原生模块）：
   npm install

2. 开发运行（热重载）：
   npm run dev
   也可双击 启动.bat（脚本内为纯英文，避免 CMD 默认 GBK 下 UTF-8 中文乱码导致命令被误解析）

3. 生产构建与运行：
   npm run build
   npm start

4. 环境变量（可选）：
   TV_PORT        端口，默认 3780
   TV_HOST        监听地址，默认 0.0.0.0（同网段手机可访问）
   TV_ADMIN_PASSWORD  管理端口令，默认 admin
   TV_SESSION_SECRET    会话密钥，生产请务必修改
   TV_DB_PATH     数据文件路径，默认 ./data/techventure.json

5. 使用流程简述：
   - 电脑连现场路由器，记下本机 IPv4（如 192.168.1.100）
   - 浏览器打开 http://本机IP:3780/admin.html ，登录后添加队伍与编号
   - 参赛手机连同一 WiFi，打开 http://本机IP:3780/play.html ，输入队伍编号
   - 大屏打开 http://本机IP:3780/screen.html
   - 每轮（顺序推进，无需填轮次 id）：管理端「保存事件」（可选）→「开放本轮提交」→ 队伍提交 →「执行本轮结算」→ 自动进入下一轮；高级里仍可手动按 id/轮次号操作
   - 活动结束后：管理端底部「重新开始一局」可清空进度并重置 4 轮（可选删队伍）；接口 POST /api/admin/activity/restart，body 需含 confirm:"RESTART"
   - 路演结束后：在管理端队伍表「+本次 / 专家」列录入追加资金与专家认可度（0–10），点「应用」；追加进入「追加池」，与基础资金合计为每轮可投入上限；结算时优先扣追加池
   - 参赛端：填写研发 / 调研 / 打磨三项（可精确到 0.1），剩余资金自动进入短期投资；短期投资按单轮 5% 利率在结算后计入可用资金（与《创想大赢家》赛事总设计 v2 一致）
   - 技术路线（LLM/KG/MM/Edge）：首场提交前可在参赛页下拉修改；一旦有任一轮提交记录则锁定
   - 排名：大屏与管理端以「加权总分」为主排序（各轮关注度按 15%/20%/25%/40% 加权），累计关注度仍为展示列
   - 调试：/debug.html 支持美式千分位显示、0.1 精度、本地保存 JSON 预设、复制 TSV

6. 模型与业务解耦：
   - 引擎：src/engine/defaultSettler.ts（创想大赢家 v2：技术路线、拥挤、领袖/跟进、关注度天花板、加权分）
   - 四科目映射：src/decision/budgetMapping.ts（短期默认不推属性，利息在 gameService 结算后入账）
   - 赛务与持久化：src/services/gameService.ts
